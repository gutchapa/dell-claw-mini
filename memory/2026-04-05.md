## Fine-Tuning vs RAG Discussion
- **Fine-tuning**: Retrains model weights permanently; knowledge baked into parameters; high cost, hard to update
- **RAG**: Retrieves relevant chunks at query time from vector DB; dynamic, cheap, easy to update; no permanent model changes
- **Hybrid approach**: Fine-tune for task/style + RAG for real-time data (best of both)

## ACP Usage Note
**CRITICAL**: When spawning ACP agents via `sessions_spawn`, must explicitly set `runtime: "acp"` parameter:
```javascript
sessions_spawn({
  runtime: "acp",  // REQUIRED for ACP routing
  task: "...",
  timeoutSeconds: 60
})
```
- Without `runtime: "acp"`, spawns default to regular subagent (kimi/kimi-code)
- With `runtime: "acp"`, routes to configured ACP agent (Pi in this case)
- Session key difference: `agent:pi:acp:...` vs `agent:main:subagent:...`
